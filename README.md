
# Interactive Project Merger

Скрипт на PowerShell с графическим интерфейсом для объединения файлов проекта в один документ (Markdown, TXT или PDF). Позволяет интерактивно выбрать, какие файлы попадут в итоговый файл, а какие — нет.

## Для чего

Быстро собрать кодовую базу проекта в один файл для:
- отправки нейросети (ChatGPT, Claude и др.) с полным контекстом
- создания архивной копии кода
- документирования структуры проекта

## Основные возможности

- **Интерактивное дерево** — видишь всю структуру проекта и решаешь, что включать.
- **4 режима на элемент**:
  - `[ OO ]` — в дереве и в слиянии
  - `[ OX ]` — только в дереве (не попадает в содержимое)
  - `[ XO ]` — скрыт из дерева, но попадает в слияние
  - `[ XX ]` — полный игнор (и дерево, и слияние)
- **Каскад на папках** — смена режима на папке применяется ко всем вложенным файлам.
- **Мультивыделение** — `Ctrl + ЛКМ` для выбора нескольких файлов, `Пробел` — переключить все выделенные сразу.
- **Управление исключениями** — прямо в окне можно добавить/удалить папки/маски (например, `node_modules`, `.git`).
- **Автоопределение** — автоматически помечает текстовые файлы как `OO`, а бинарные как `OX`.
- **Выбор формата** — Markdown с подсветкой кода, Plain Text или PDF (через Python).
- **Кастомный путь** — можно указать любую папку прямо в окне, если запустил без параметров.

## Требования

- **Windows** 10/11
- **PowerShell** 5.1 или 7.x
- **eza** — современная замена `tree`/`ls` для построения красивого дерева каталогов

## Установка eza

Если `eza` не установлена, скрипт выдаст ошибку при запуске.

**Через winget** (рекомендуется):
```powershell
winget install eza-community.eza
```

**Через scoop**:
```powershell
scoop install eza
```

После установки перезапусти терминал.

## Запуск

```powershell
# Текущая папка
.\interactive_merge.ps1

# Конкретная папка
.\interactive_merge.ps1 "C:\Projects\MyApp"
```

В открывшемся окне выбери режимы для файлов и папок, затем нажми **«Собрать»**.
